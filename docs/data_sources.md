# 海況・気象データソース一覧 (Data Source Inventory)

AI予測モデルに使用している各データの格納先、一次ソース、および取得状況のまとめです。

| 項目グループ | 具体的な項目名 | 格納先DBテーブル | **データの一次ソース (Origin)** | 頻度と現状 |
| :--- | :--- | :--- | :--- | :--- |
| **気象・陸上** | 気温、風速、降水量、日照時間 | `weather_history` | **Open-Meteo API** | 日次で充足。 |
| **潮汐** | 潮回り（大潮〜若潮）、潮位 | `tide_history` | **AppSync API (独自システム)** | 日次で充足。 |
| **波浪** | 波高、波向き | `openmeteo_marine_history` | **Open-Meteo API (Marine)** | 日次で充足。APIから直接取得。 |
| **河川** | 河川流量（荒川等） | `openmeteo_marine_history` | **Open-Meteo API (Marine)** | 日次で充足。APIから直接取得。 |
| **海流フラグ** | 黒潮大蛇行フラグ | (計算ロジック) | 海上保安庁等の発表定義 | 固定値。 |
| **水質 (重要)** | 海面水温 (SST) | `tokyo_bay_marine_data` | **東京湾環境情報センター (TBEIC)** | **各観測塔から日次で自動同期。** |
| **水質 (重要)** | 塩分濃度、DO、流向・流速 | `tokyo_bay_marine_data` | **東京湾環境情報センター (TBEIC)** | **各観測塔から日次で自動同期。** |
| **水質 (参考)** | 透明度、COD等 | `tokyo_bay_marine_data` | 千葉県環境生活部 (補完用) | 不定期更新。 |

## データ取得の仕組み
- **TBEIC 同期**: `scripts/sync_tbeic_marine_data.py` により、東京湾内の主要な観測塔（中ノ瀬、本牧等）の連続観測データを取得。
- **Open-Meteo 直接統合**: `src/ml/generate_calendar.py` 等で Marine API を直接叩き、予測に利用。

## 今後の改善・検討事項
- **データ欠損への対応**: 観測塔のメンテナンス等による欠損時の補完ロジックの強化。
- **予測精度のフィードバック**: 取得した実測データを用いた、過去の予測値の精度検証パイプラインの構築。
